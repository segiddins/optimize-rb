# post.md → Slidev Deck Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.
>
> **Note on execution model:** This plan is inherently sequential with user-in-the-loop review between paragraph decompositions. Subagent parallelism is not a good fit; inline execution with the user reviewing each section's beat decomposition is the intended mode.

**Goal:** Turn finalized `post.md` into a RubyKaigi 2026 Slidev deck in ≤1 hour, with every source paragraph mapped to a slide cluster and preserved verbatim as speaker notes.

**Architecture:** Slidev source at `talk/slides/slides.md`, committed in-repo. Per-section workflow: structural split → beat decomposition (with `v-click` reveals and code-line highlights) → paste source paragraph into notes. Review-as-you-go per section, not per slide.

**Tech Stack:** Slidev (Vue-based, Markdown-authored slides), Node/npm, git/jj.

**Related spec:** `docs/superpowers/specs/2026-04-24-post-to-slides-plan.md`

- **Create:** `talk/slides/slides.md` — the Slidev source file. Single file; Slidev's per-slide separator is `---`.

- **Read-only references:** `post.md` (source prose), `talk/references.md` (citations), `docs/yarv-instructions.md` (YARV decoder).

One file for all slides. Slidev supports per-slide frontmatter inline; no need for a multi-file setup at this length.
